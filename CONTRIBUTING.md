# Contributing to npllm

Thank you for your interest in contributing to npllm! This document provides guidelines for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/npllm.git`
3. Create a virtual environment and install dependencies
4. Create a branch for your changes: `git checkout -b feature/your-feature`

## Development Setup

See [Quick Start Guide](docs/QUICK_START.md) for detailed setup instructions.

## Code Style

- Follow PEP 8 for Python code
- Use type hints where possible
- Add docstrings to all public functions/classes
- Keep functions focused and small

## Testing

Run tests before submitting:

```bash
pytest tests/
```

Add tests for new features.

## Documentation

- Update relevant documentation files
- Add examples for new features
- Keep documentation in English
- Use clear, concise language

## Commit Messages

Follow conventional commits:

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `refactor:` Code refactoring
- `test:` Adding tests
- `chore:` Maintenance tasks

Example: `feat: Add streaming support to API`

## Pull Requests

1. Ensure all tests pass
2. Update documentation
3. Write clear commit messages
4. Reference issues if applicable
5. Request review from maintainers

## Reporting Issues

Use GitHub Issues to report bugs or request features. Include:

- Description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, Python version, etc.)
- Relevant logs

## Questions?

Open a discussion on GitHub or contact maintainers.
